# Review Command

Review the current working tree or requested diff.

Prioritize findings in this order:

1. Bugs or behavioral regressions.
2. Security, privacy, or destructive-operation risk.
3. Provider-neutrality problems.
4. Missing tests for changed behavior.
5. Maintainability concerns caused by the current change.

Lead with findings and file references. Keep summaries secondary.
