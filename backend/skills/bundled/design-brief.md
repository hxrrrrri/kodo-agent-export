# Design Brief

Use this skill to produce structured design brief documents that align creative teams and clients before work begins. Output is a professional HTML or Markdown document.

## Brief Structure

```
1. Project Overview
2. The Brand / Context
3. Target Audience
4. Project Goals & Success Metrics
5. Visual Direction
6. Competitive Landscape
7. Deliverables
8. Constraints & Requirements
9. Timeline
10. Stakeholders & Approvals
11. Open Questions
```

## Project Overview Section

Answer these in 2–3 sentences each:
- What are we making?
- Why are we making it now?
- What problem does it solve for the audience?

## Brand Context

If for an existing brand:
- Brand personality adjectives (3–5 descriptors)
- What the brand sounds like (formal, playful, authoritative, approachable)
- What the brand should NOT be
- Reference designs that are on-brand
- Reference designs that are explicitly off-brand

If for a new brand:
- Inspiration references (competitors to emulate, styles to explore)
- Personality and voice brief
- Colors to explore / avoid

## Audience Profile

```markdown
### Primary Audience
**Who**: [Job title, role, or persona name]
**Age range**: XX–XX
**Context**: Where do they encounter this design? (browsing, print, in-product)
**Goals**: What do they want to accomplish?
**Pain points**: What frustrates them today?
**Sophistication**: Technical novice / intermediate / expert

### Secondary Audience (if applicable)
…
```

## Visual Direction Section

Choose or specify:

| Dimension | Options |
|-----------|---------|
| Fidelity | Wireframe / High-fidelity / Production-ready |
| Style | Minimal / Bold / Editorial / Playful / Technical |
| Mood | Professional / Warm / Energetic / Calm / Premium |
| Color approach | Monochrome / Brand colors / Neutral + accent / Full palette |
| Typography | Serif / Sans-serif / Mixed / Monospace |
| Imagery | Photography / Illustration / Icons only / Data-driven |

## Deliverables Table

| Deliverable | Format | Quantity | Notes |
|-------------|--------|----------|-------|
| Landing page | HTML / Figma | 1 | Desktop + mobile |
| Email template | HTML | 3 variants | Promotional, transactional, onboarding |
| Brand assets | SVG, PNG | Logo + 4 icons | Multiple colorways |

## Constraints & Requirements

**Hard constraints** (non-negotiable):
- Must use existing brand colors: [colors]
- Must include legal disclaimer in footer
- Accessibility: WCAG 2.1 AA minimum

**Soft constraints** (prefer but can discuss):
- Prefer to avoid hero videos (performance concern)
- Prefer single-column mobile layout

**Out of scope** (explicitly excluded):
- Animation beyond hover states
- Backend integration
- Multi-language versions (Phase 2)

## Timeline Format

```markdown
| Milestone | Date | Owner |
|-----------|------|-------|
| Brief sign-off | [Date] | Client |
| Initial concepts (3 directions) | [Date] | Designer |
| Client feedback due | [Date] | Client |
| Revised direction | [Date] | Designer |
| Final approval | [Date] | Client |
| Delivery | [Date] | Designer |
```

## Writing Standards

- Be specific: "modern" means nothing → "Linear/Vercel-style with monochrome surfaces and one accent color"
- Constraints are hard or soft — mark them clearly
- All dates are absolute (no "in 2 weeks")
- Deliverables table includes format, quantity, and notes
