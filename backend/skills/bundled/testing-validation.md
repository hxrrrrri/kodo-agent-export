# Testing and Validation

Close the loop with targeted tests and reproducible verification.

## Steps
- Add tests for new command paths, API routes, and tool safety behavior.
- Prefer focused unit tests for edge cases and contract tests for endpoints.
- Run backend and frontend checks, then report failures with root cause.
- Include manual smoke checks for any feature that depends on streaming UI.

## Output
- New tests added and what they prove.
- Validation command results.
- Remaining gaps and exact follow-up tasks.
