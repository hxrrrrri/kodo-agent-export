# Web Research

Gather fresh external context before making implementation decisions.

## Steps
- Start with `/search <query>` for current docs, changelogs, or API behavior.
- Compare at least two sources when behavior is ambiguous.
- Extract only actionable facts and convert them into concrete implementation notes.
- If sources conflict, call out uncertainty and choose the safer path.

## Output
- A concise set of verified facts.
- The code or config changes these facts imply.
- Any unresolved assumptions that need user confirmation.
